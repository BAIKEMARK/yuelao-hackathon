# 《红绳懒得系，弹珠自己撞》H5 实施计划

## 1. 目标

实现一个手机端优先的 AI H5 MVP：

```text
开始界面 -> 选母球 -> 封闭弹珠盘发射一次 -> 统计碰撞 -> 锁定 CP -> 阶跃 AI 生成姻缘档案 -> 存入 CP 图鉴
```

本计划按真实 AI 接口版本推进，不做离线故事兜底。

## 2. 技术选型

- 应用目录：`弹珠姻缘/`
- 框架：Next.js + React
- 样式：CSS Modules 或全局 CSS，优先移动端竖屏
- 弹珠盘：Canvas + Matter.js
- AI 接口：StepFun `step-3.7-flash`
- API 文档：`https://platform.stepfun.com/docs/zh/guides/models/step-3.7-flash`
- API Key：读取根目录 `.env` 中的 `API_KEY`
- API 代理：Next.js server route，前端只请求本项目 `/api/generate-story`
- 图鉴存储：浏览器本地持久化，key 建议为 `marble_cp_gallery_v1`
- 贴图策略：先用文字、色块和圆形占位，后续替换为背景图和角色球贴图

## 3. 文件结构

计划新增：

```text
弹珠姻缘/
  package.json
  next.config.mjs
  app/
    layout.jsx
    page.jsx
    globals.css
    api/
      generate-story/
        route.js
  components/
    StartScreen.jsx
    CharacterSelect.jsx
    MarbleBoard.jsx
    ResultCard.jsx
    Gallery.jsx
  lib/
    characters.js
    storyPrompt.js
    galleryStorage.js
    stepfunClient.js
  test/
    pairing.test.mjs
    storyPrompt.test.mjs
```

职责：

- `app/page.jsx`：管理主流程状态。
- `components/StartScreen.jsx`：开始页和图鉴入口。
- `components/CharacterSelect.jsx`：选择 1 个母球，剩余 9 个进入弹珠盘。
- `components/MarbleBoard.jsx`：Canvas/Matter.js 物理盘、拖动瞄准、单次发射、倒计时、碰撞统计。
- `components/ResultCard.jsx`：展示 AI 姻缘档案，提供存入图鉴和再撞一次。
- `components/Gallery.jsx`：展示本地保存的 CP 档案。
- `lib/characters.js`：10 个初始角色数据。
- `lib/storyPrompt.js`：构建阶跃模型输入 prompt。
- `lib/galleryStorage.js`：封装 localStorage 读写。
- `lib/stepfunClient.js`：服务端调用 StepFun，不在前端导入。
- `app/api/generate-story/route.js`：后端代理，读取 root `.env` 的 `API_KEY`。
- `test/*.mjs`：验证配对规则和 prompt 结构。

### Task 3：弹珠盘与碰撞统计

目标：实现封闭盘面、单次发射、倒计时和胜出规则。

步骤：

1. 在 `MarbleBoard.jsx` 初始化 Matter.js engine、render loop 和封闭边界墙。
2. 将母球和 9 个目标角色生成为圆形 body。
3. 拖动母球时显示方向和力度，松手后只发射一次。
4. 监听碰撞事件，只统计母球与目标球的直接碰撞。
5. 倒计时结束后停止统计。
6. 结果选择规则：

```js
function resolveWinner(hitCounts, lastHitId) {
  const entries = Object.entries(hitCounts);
  const max = Math.max(...entries.map(([, count]) => count));
  const tied = entries.filter(([, count]) => count === max).map(([id]) => id);
  return tied.length === 1 ? tied[0] : lastHitId;
}
```

7. 验证：

```bash
npm test
```

测试覆盖：

- 最高碰撞次数唯一时取最高者。
- 并列时取 `lastHitId`。
- 没有二次发射入口。
- 球体位置异常时被拉回盘内合法区域。

### Task 4：AI 故事生成代理

目标：通过服务端 route 调用 StepFun `step-3.7-flash`。

步骤：

1. 创建 `lib/storyPrompt.js`，输入母球、目标角色、碰撞次数和是否“命运最后一撞”，输出 messages。
2. 创建 `lib/stepfunClient.js`，只在服务端使用。
3. 在 `app/api/generate-story/route.js`：
   - 从 `../../.env` 对应的项目根上级路径读取 `API_KEY`。
   - 调用 `https://api.stepfun.com/v1/chat/completions`。
   - model 使用 `step-3.7-flash`。
   - 返回 JSON：`title`、`relationshipTag`、`story`、`yuelaoComment`。
4. 前端等待接口返回期间展示生成中状态。
5. 验证：

## 5. 开发风险

- StepFun 文档页面是动态站点，实际字段以接口调通结果为准。
- 根目录 `.env` 不在 Next.js app 目录内，API route 需要显式加载上级 `.env`。
- Matter.js 在移动端需要处理触摸事件和 canvas 尺寸缩放。
- 贴图后续替换时，角色球碰撞体尺寸不能随图片原始尺寸漂移。
- AI 响应可能不是严格 JSON，需要在实现中要求模型输出 JSON，并保留解析失败时的错误提示。
