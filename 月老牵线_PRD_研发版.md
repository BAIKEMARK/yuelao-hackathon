# 《姻缘簿：月老牵线局》PRD（团队研发版）

## 1. 文档目标

本文档用于指导黑客松第一阶段研发分工，重点定义：

- 第一阶段 MVP 范围。
- 本地互动空间版与 AI API 完整版的交付边界。
- 两版共用的数据协议。
- 页面流程、玩法规则、数值规则。
- “林黛玉 × 孙悟空”完整样板线的内容规格。

本文档不负责路演包装，不展开商业化、长期运营和复杂扩展系统。

## 2. 第一阶段范围

### 2.1 必做范围

第一阶段必须完成一个可体验闭环：

```text
接案台
  -> 林黛玉委托详情
  -> 候选池选择
  -> 林黛玉 × 孙悟空三回合相亲
  -> 姻缘簿结果记录
```

必做能力：

- 接案台展示 2-3 个委托卡片，仅林黛玉完整开放，其余为占位或待解锁。
- 林黛玉委托详情页展示人物需求与姻缘簿命格卡。
- 候选池展示 4-5 个候选对象，仅孙悟空完整可玩，其余简版或待解锁。
- 三回合相亲，每回合选择地点、事件、话题。
- 每回合展示对话、内心 OS、数值变化、月老 KPI 反馈。
- 结尾展示关系档次、月老称号、相亲过程记录。

### 2.2 双版本并行交付

| 版本 | 目标 | 实现方式 | 约束 |
| --- | --- | --- | --- |
| 本地互动空间版 | 稳定提交与现场体验 | 本地分支、预写文案、规则数值、模板结果 | 8MB 内、无网络、入口 `index.html` |
| AI API 完整版 | 展示 AI 核心玩法 | LLM 生成对话、OS、结尾记录；规则引擎算数值 | 可联网，不受互动空间提交约束 |

两版可以分开实现 UI，但必须遵守同一套数据协议，保证内容和玩法语义一致。

### 2.3 暂不做范围

- 不做真实账号、存档、登录。
- 不做多人社交。
- 不做复杂角色养成。
- 不做全候选对象完整路线。
- 不做大量图片素材或复杂角色动画。
- 不让 LLM 决定核心数值，避免不可控。

## 3. 页面流程

### 3.1 接案台

目标：让用户理解自己是月老，正在接收名著人物的情感委托。

内容：

- 2-3 张委托卡片。
- 林黛玉卡片可点击进入。
- 其他卡片显示“命簿未开 / 待解锁 / 后续委托”。

林黛玉卡片建议文案：

```text
委托人：林黛玉
委托摘要：我想见一个不怕我说真话、也听得懂我话中意思的人。
当前状态：可接案
```

### 3.2 委托详情

目标：展示求助者需求、人物属性、玩法目标。

内容：

- 委托原文。
- 姻缘簿命格卡。
- 关系目标说明：尽量促成高质量关系，但不强求爱情。
- 进入候选池按钮。

### 3.3 候选池选择

目标：让用户完成“月老选择撮合对象”的关键动作。

候选对象：

| 候选 | 状态 | 展示说明 |
| --- | --- | --- |
| 孙悟空 | 完整可玩 | 反差强，表达轻，承诺重 |
| 贾宝玉 | 简版 / 待解锁 | 熟人局，情绪共鸣强 |
| 唐僧 | 简版 / 待解锁 | 稳定克制，信任高，心动慢 |
| 武松 | 简版 / 待解锁 | 安全感强，表达笨拙 |
| 猪八戒 | 简版 / 待解锁 | 热情直接，轻喜剧风险高 |

第一阶段点击孙悟空进入完整相亲。点击其他候选时可展示简短提示，不进入完整局。

### 3.4 三回合相亲

每回合包含：

1. 展示当前回合主题。
2. 用户选择地点。
3. 用户选择事件。
4. 用户选择话题。
5. 系统展示角色对话与内心 OS。
6. 系统展示心动 / 信任 / 默契 / KPI 变化。
7. 写入本局过程记录。

### 3.5 姻缘簿结果记录

内容：

- 最终关系档次。
- 三维关系值。
- 月老 KPI 与称号。
- 三回合选择摘要。
- 关键角色反应。
- 一段“相亲过程记录”。
- 重新牵线按钮。

## 4. 核心玩法规则

### 4.1 回合结构

第一阶段固定三回合。

| 回合 | 选项密度 | 玩法作用 |
| --- | --- | --- |
| 第 1 回合 | 地点 2、事件 2、话题 2 | 建立第一印象 |
| 第 2 回合 | 地点 2、事件 2、话题 2 | 制造互动与误会 |
| 第 3 回合 | 地点 3、事件 3、话题 3 | 推动关系定局 |

### 4.2 数值维度

核心关系值：

- `romance`：心动。
- `trust`：信任。
- `chemistry`：默契。

月老表现值：

- `kpi`：月老 KPI。

数值范围建议为 `0-100`。初始值由配对关系决定。

### 4.3 关系档次判定

建议第一阶段使用规则判定，具体阈值可在调试中微调。

| 档次 | 条件建议 | 说明 |
| --- | --- | --- |
| S 爱情 / 正缘牵成 | 心动 >= 75，信任 >= 65，默契 >= 65 | 最高档，红线牵成 |
| A 欢喜冤家 / 暧昧拉扯 | 心动 >= 70，默契 >= 55，信任 >= 45 | 心动强，冲突仍在 |
| B 知己 / 灵魂互补 | 信任 >= 70，默契 >= 70，心动 < 75 | 互相理解，但未必恋爱 |
| C 礼貌收场 | 三项均未严重低于 40 | 没有翻车，也未深入 |
| D 红线打结 / 强扭翻车 | 任一核心值 < 30 或 KPI < 30 | 明显不自然 |

判定优先级建议：

```text
D 翻车
  -> S 爱情
  -> A 欢喜冤家
  -> B 知己
  -> C 礼貌收场
```

## 6. 本地版与 AI 版生成边界

### 6.1 本地互动空间版

本地版使用：

- 预写 `RoundOption`。
- 预写关键组合的 `dialogue` 和 `innerThoughts`。
- 通用模板兜底非关键组合。
- 固定规则计算 `statDelta` 与 `kpiDelta`。
- 模板拼接 `finalStory`。

本地版可以通过 UI 表现模拟生成感：

- 打字机效果。
- “月老正在翻姻缘簿”加载态。
- 内心 OS 气泡延迟弹出。

### 6.2 AI API 完整版

AI 版使用：

- 同一套角色卡、选项、当前数值、回合历史作为 prompt 输入。
- LLM 生成 `dialogue`、`innerThoughts`、`summary`。
- 结尾 LLM 生成 `finalStory`。
- `statDelta`、`kpiDelta` 和 `endingId` 仍由规则引擎产生。

AI 版每回合推荐输入：

```text
角色卡：林黛玉、孙悟空
当前关系值：心动 / 信任 / 默契
月老 KPI
本回合选择：地点、事件、话题
历史回合摘要
输出要求：2-4 句对话、2 条内心 OS、1 句回合摘要
```

## 7. 林黛玉 × 孙悟空样板线

### 7.1 配对前提

求助者：林黛玉。

委托：

```text
我想见一个不怕我说真话、也听得懂我话中意思的人。
```

候选对象：孙悟空。

配对张力：

- 林黛玉敏锐、清醒、骄傲，不愿为了被爱而变得好相处。
- 孙悟空听不懂弦外之音，但愿意认真学。
- 林黛玉说真话，孙悟空不怕真话。
- 孙悟空表达不细，林黛玉能逐渐看见他行动里的承诺。

路线气质：

- 反差喜剧：鸡同鸭讲、误会、直球回应。
- 治愈成长：不是拯救，而是互相学习对方的表达方式。

### 7.2 初始数值

```ts
initialStats = {
  romance: 35,
  trust: 40,
  chemistry: 30
};

initialKpi = 60;
```

### 7.3 第 1 回合：初见

回合目标：建立第一印象。

地点选项：

| id | 标签 | 倾向 |
| --- | --- | --- |
| `r1_place_flower_path` | 大观园花径 | 更适合林黛玉，提升信任 |
| `r1_place_heaven_gate` | 南天门外 | 更适合孙悟空，提升新鲜反差 |

事件选项：

| id | 标签 | 倾向 |
| --- | --- | --- |
| `r1_event_fallen_flower` | 落花被风卷走 | 触发林黛玉表达，考验孙悟空能否接住 |
| `r1_event_monkey_noise` | 猴群误入相亲局 | 喜剧感强，可能提升默契或降低信任 |

话题选项：

| id | 标签 | 倾向 |
| --- | --- | --- |
| `r1_topic_truth` | 说真话会不会得罪人 | 贴合林黛玉委托，提升理解空间 |
| `r1_topic_freedom` | 自由是不是不受管束 | 贴合孙悟空，可能触发价值观碰撞 |

### 7.4 第 2 回合：试探

回合目标：制造互动、误会和补救空间。

地点选项：

| `r2_place_peach_garden` | 蟠桃园边 | 让林黛玉进入孙悟空的世界 |

事件选项：

| id | 标签 | 倾向 |
| --- | --- | --- |
| `r2_event_misread_poem` | 孙悟空把诗听岔了 | 喜剧冲突，后续补救可升默契 |
| `r2_event_protective_action` | 孙悟空直接替她挡下一场闹剧 | 行动承诺强，提升信任 |

话题选项：

| id | 标签 | 倾向 |
| --- | --- | --- |
| `r2_topic_understanding` | 什么才算听懂一个人 | 治愈成长倾向 |

### 7.5 第 3 回合：定局

回合目标：将关系推向爱情、知己、欢喜冤家或收场。

## 8. 角色命格卡内容

### 8.1 林黛玉

```text
心结：不愿被当成需要小心哄着的人，也不愿自己的真话被说成多心。
情感需求：有人能听懂她的锋芒和话中意思，并愿意平等接招。
表达方式：敏锐、含蓄、带刺，但不是无理取闹。
雷区：敷衍、装懂、把她的敏锐当成脆弱。
加分点：认真听她说完，正面回应，不用廉价安慰压住她。
月老备注：别急着哄，她不是玻璃，她是会开口的刀光。
```

### 8.2 孙悟空

```text
心结：不擅长听弦外之音，也不习惯把承诺说得细软。
情感需求：有人看见他的赤诚，而不是只看见闹腾和不受管。
表达方式：直来直去，先行动后解释。
雷区：被要求装乖、被误解为不可靠、被绕太久还不给明话。
加分点：给他明确问题，让他用行动证明。
月老备注：嘴上像风火轮，承诺压得住五指山。
```

## 9. KPI 与称号

### 9.1 KPI 规则

KPI 用于反馈月老操作质量，不作为强失败门槛。

增加 KPI 的情况：

- 选择符合双方情感需求。
- 让误会得到补救。
- 推动两人互相理解。
- 触发高质量对话或名场面。

降低 KPI 的情况：

- 持续踩角色雷区。
- 强推不自然话题。
- 制造冲突后不给补救空间。

### 9.2 月老称号

| 条件 | 称号 |
| --- | --- |
| S 结局且 KPI 高 | 金牌红娘 |
| A 结局 | 欢喜局高手 |
| B 结局 | 知己牵线人 |
| C 结局 | 礼貌收场师 |
| D 结局 | 红线打结师 |
| KPI 低但心动高 | 误会制造机 |

## 10. 姻缘簿结果记录

### 10.1 结构

结果页必须包含：

- 委托人。
- 候选对象。
- 最终关系档次。
- 月老称号。
- 心动 / 信任 / 默契最终值。
- 三回合选择记录。
- 每回合一句关键回顾。
- 最终相亲过程记录。

### 10.2 本地版模板

```text
本局姻缘簿记录：

月老为【林黛玉】选中了【孙悟空】。
初见时，他们在【地点1】因【事件1】谈起【话题1】。
第二回合，二人在【地点2】遇到【事件2】，关系出现【回合2摘要】。
最后，月老安排【地点3】与【话题3】，让这段红线落在【最终关系】。

最终判词：【根据 endingId 选择模板】
```

### 10.3 AI 版生成要求

AI 版结果记录必须：

- 引用玩家三回合真实选择。
- 保持林黛玉和孙悟空的人物口吻。
- 不把林黛玉写成等待拯救的弱者。
- 不把孙悟空写成只会莽撞的笨人。
- 结尾明确关系档次和月老称号。

## 11. 第一阶段验收标准

### 11.1 本地互动空间版

- 根目录包含可运行入口 `index.html`。
- 不依赖外部网络或 CDN。
- 包体小于 8MB。
- 能完整跑通一局林黛玉 × 孙悟空。
- 三回合均有选择、反馈、数值变化。
- 结果页能生成相亲过程记录。

### 11.2 AI API 完整版

- 能使用同一套角色卡和选项协议。
- 每回合能调用 LLM 生成对话和内心 OS。
- 数值变化仍由规则控制。
- 结尾能生成包含三回合选择的姻缘簿记录。
- 当 API 失败时，应能降级到本地模板或错误提示，不阻断演示。

### 11.3 内容验收

- 林黛玉不被写成“需要被拯救的脆弱角色”。
- 孙悟空不被写成单纯莽撞。
- 两人关系从反差开始，但能走向理解。
- 爱情是最高档，但知己、欢喜冤家等结局也成立。
