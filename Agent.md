# Agent Notes

## Project Context

This workspace is for a hackathon entry under “赛道一｜互动空间”. The goal is to identify and build a lightweight interactive work that can be experienced directly, preferably within 24 hours of development time.

The current priority is not implementation yet. The team is still searching for a concept that is genuinely fun, immediately understandable, and feasible under the platform constraints.

## Hard Constraints

- Interactive Space demo should be under 8MB.
- Demo should not rely on network communication.
- Entry file must be `index.html`.
- Prefer a single-file or minimal-file HTML project.
- Avoid concepts that require many image assets, character animations, cutout AI images, or heavy art production.
- If a full version uses AI APIs, prepare an offline demo fallback.

## Design Priorities

Prioritize:

- Immediate fun over decorative concept.
- A clear core action users understand within seconds.
- Replayability or a short challenge loop.
- Interaction that has feedback during play, not only at the result screen.
- Low-asset visual systems: chat UI, form UI, notification UI, typography, particles, simple Canvas, procedural shapes.
- If using AI, make AI part of the core interaction, not just a final text generator.

Avoid:

- Cold metric reports that feel like generic personality tests.
- Result cards that users would not actually want to share.
- Overly “weird” premises that require explanation before they sound fun.
- Traditional game concepts that need lots of art resources within 24 hours.
- Pure CSS/SVG character scenes that risk looking rough.

## Rejected Or Deprioritized Ideas

The following ideas are currently deprioritized:

- 课桌洞大搜查
- 小卖部冲刺
- 一掌拍翻童年
- 作业补完模拟器
- 橡皮大战

Other ideas such as “AI 眼里的你”, “五句话套出密码”, and “别说关键词” have partial merit but need stronger core fun before being selected.

## Current Open Question

Find a concept that satisfies all of the following:

1. It is easy to understand at first glance.
2. The interaction itself is fun, not just the final generated output.
3. It can be built in 24 hours without heavy assets.
4. It can optionally use AI in the full version.
5. It has a credible reason for users to replay or invite others to try.
